LLM + RL

LM이 해야하는 역할은 목표 설정이라고 생각함.

-> 목표는 지도가 없는 에이전트의 입장에서 지도를 스스로 그리면서 지식을 넓혀가는 것. 전제는 RL이 주어진 역할을 잘 한다는 것.

랜드마크형 중심(POI)의 토폴로지 그래프를 만들면 탐험에 조금 더 도움이 되지 않을까? 워프 포인트 - 모든 지역(집, 1번도로, 상록시티, 오박사 연구실 etc..) 노드를 추가할 때에는 건물 ID나 map ID에 대한 text 매핑 정보 존재해야지 text로 적용이 가능할 것임. --> 현재 존재하는 map_data.json이 있음.

전반적인 아이디어

시작 할 때마다 Dual-Node 지도를 초기화 함. <- 에피소드가 리셋될 때 마다. Dual Node 지도는 에이전트가 이동한 위치를 저장하는 용도이자, LLM이 목표를 줄 수 있는 지표이다.

첫번째 지도 (Directed Graph 기반)

why? 두 마을이 양방향으로 뚫려있는 구조가 아니기 때문. 현재 노드에서 새로운 노드로 넘어갈 시 prev node와 current node로 구분하여 edge를 연결함. - 태초마을, 1번도로, 상록시티 같은 큰 지역을 담을 수 있는 것

두번째 지도

  • 태초마을의 내부, 1번 도로의 내부를 볼 수 있는 세부 지역의 지도, NPC같은 정보를 노드로 담는 것이 아니라, 포켓몬 센터, 포켓마켓, 오박사 연구실 등의 작은 지역이나 위치에 대한 정보를 담음. (NPC 정보는 담기 번거롭고 map_data.json같은 사전 정보가 존재하지 않아서 LLM이 활용하기 어렵다고 생각)

지도를 채우는 방법

  • pokemon_puffer에 이미 map_data.json 현재 map_id와 local x, y좌표를 통해 매핑할 수 있는 코드가 작성되어 있음. (들어가는 곳과 나가는 곳을 모두 저장해야 함)
  • 이 정보를 토대로 현재 위치에 대한 맵 Name, 맵 id, local, global 좌표, 인접 노드 최소 거리 리스트(희망 사항)이 존재함. 첫번째 지도에 노드에는 두번째 지도에 대한 격자배열 주소가 존재해야 함.

에이전트는 새로운 지역에 도착하였을 때, 두번째 지도를 채우는 것이 1번 목표가 됨. 두번째 지도는 (n*m) 격자지도.

작은 지도에서는 2D 행렬로 확장하는 것으로 함. 작은 지도는 현재 맵에 존재하는 위치 정보를 리스트로 가지고 있음 예를 들면, ["PC", "PM",... 등등] 실제 칸수 지도로써 이동한 칸을 늘리면서 각 칸에 정보를 남김 예를 들면, 그냥 아무 상관없는 타일이라면 "" string으로 비워두고, 나머지는 map_data.json의 이름을 따르되, 입구와 출구를 명확히 구분 해야함.

LLM의 역할

"Event flag가 변경되었을 때" 호출하는 방식이고 Dual-Node의 지도를 토대로 Name을 목표를 줌.

큰 지도에서

예를 들어, 현재 특정지역 포켓몬 라이벌 전투가 발생하였다는 정보가 주어지면, LLM은 다음 해야할 일을 알 것이고 그 위치를 map_data.json에서 찾아 뽑아냄. 그리고 현재 위치와 비교하여 "지도에 존재하는 경유 지역들을 위치이동 RL에게 넘김" 그러나, 목표 지점이 현재 에이전트가 탐색하지 못한 지점일 가능성도 존재함. 이 경우에는 LLM이 기존 지도를 확인한 후 목표와 가장 가까운 지역들을 위치이동 RL에게 넘기고, flag로써 탐색이 필요함을 알림.

->이 위치 Name 텍스트를 받은 RL 정해진 지역으로 갈 수 있는 RL로, 정보를 토대로 이동함. Name 텍스트에 해당하는 위치가 보상위치로 정해지는 느낌이면 가능할 것 같음. !!!RL이 과연 갑자기 주어진 텍스트를 통해서 이동을 할 수 있을 것인가가 문제 -> 보상이 주어진다고 해도 어떻게 바로 알 수 있나?

-> 탐색 RL은 가장 가까운 지역에 도달하였을 때, 기존의 목표 지역을 찾고자 함.

큰 지도에서 목표 지점에 도착한 에이전트는 1차적으로 작은 지도의 위치 정보 리스트를 보면서 목표가 있는 지 확인 함. 없을 시에는 탐색 RL로 맵을 확장하는 것을 목표로 하고, 목표를 발견할 때까지 새로운 공간 탐색을 유도함.

RL의 역할

LLM은 무엇을 할 수 있을까 어디로 가야하는 지의 목적지를 사용할 수 있을 것 같다... RL의 사용 -> 지도를 넓힐 목적의 RL이랑 정해진 지역으로 갈 수 있는 RL로 나눔 <-- 얘를 어떻게 만들 수 있을 지가 중요할 듯, 상호작용 RL은 NPC 위주 현재 위치에서 상호작용을 선호하는 RL 3가지 사용. 전투는 배제. 특정 RL을 사용하는 것이 아니라 RL들이 할 수 있는 것

탐색 RL 현재 있는 위치에서 새로운 공간을 발견하고자 할 때 사용하는 RL

위치이동 RL 목표 위치로 이동하고자 할 때 사용하는 RL
